# Apple's Priority Stack

## Definition

The order in which Apple resolves conflicts when stakeholder interests don't align:

1. **Apple's own needs**
2. **Users**
3. **Developers**

When user and developer interests conflict, Apple sides with users (Cook stated this under oath in the Epic trial). When Apple's own interests conflict with either, Apple wins — and Apple does not like to talk about this layer publicly.

The stack is a useful predictive tool: any time you're trying to forecast Apple's behavior on a contested platform decision, walk down the stack and find the first level where the answer becomes determinate.

## Origin

- John Gruber, *Daring Fireball* (2021) — articulated the three-tier ordering and the Apple-first qualifier.
- Tim Cook, Epic Games trial testimony — confirmed users-over-developers on the record.

## Examples across articles

- [2026-05-04 Tim Cook's Impeccable Timing](../articles/2026-05-04-tim-cooks-impeccable-timing-stratechery-by-ben-thompson.md) — Ben quotes Gruber's framing to interrogate App Store policy. Predictions the stack makes:
  - **App Tracking Transparency (2021)** — users (privacy) over developers (ad targeting). Stack predicts this; Apple chose users.
  - **30% App Store cut, refusal to lower it** — Apple over developers. Schiller proposed lowering it in 2011 once profits hit $1B; Cook didn't follow the advice. Stack predicts this.
  - **Gruber's worry** — that under-investing in developer goodwill eventually erodes the *Apple-first* layer too, because alienated developers are a long-term Apple liability. Suggests the stack has a hidden self-undermining property if level 1 ignores levels 2/3 too long.
